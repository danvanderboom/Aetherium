## Slice B — Configurable player death loop + client surface

- [x] B.1 `Downed` component (player down-state with its own tick countdown, `Aetherium.Server.Combat.Downed`), distinct from monster `Dying`. Also added `RespawnInvulnerable` (a second, short-lived component for the post-respawn protection window) and `PlayerDeathResolver` (a pure `Permadeath × DownStateEnabled → outcome` mapping, unit-tested in isolation like `DeathPolicy.ResolveDyingTicks`).
- [x] B.2 `GameMapGrain.ResolvePlayerLethalHit`: on a lethal monster→player hit, applies `PlayerDeathResolver`'s outcome — instant respawn / enter `Downed` / → `Corpse` (permadeath)
- [x] B.3 `StepNpcsAsync`'s `AttackOutcome` handling now checks `RemainingHealth <= 0` + target-is-a-player and routes to B.2 instead of the old plain-health-delta path; non-lethal hits and monster-vs-monster hits are unaffected
- [x] B.4 `RespawnPlayer`/`ResolveRespawnLocation` implement `RespawnLocationPolicy`: `DeathLocation`/`EntryLocation`/`FixedCoordinates`/`OffsetFromCoordinates` resolve directly (with an occupancy check that correctly excludes the player's own entity, so `DeathLocation` doesn't always reject "in place"); `WorldSpawn` and every mode needing an unbuilt subsystem (`NamedLocation`/`OffsetFromNamedLocation`/`LastSafeLocation`/`PartyLeader`) fall back to `SelectUnusedSpawn()`. Reconciles `_spawnsInUse`/`_playerSpawns`, emits `EntityMovedDelta` + Health delta, applies `RespawnInvulnerabilityTicks` via `RespawnInvulnerable`.
- [x] B.5 `IsActionable` gate added to all 8 player-command method bodies (`MoveAsync`, `RotateAsync`, `ChangeLevelAsync` directly; `PickupAsync`/`DropAsync`/`AttackAsync`/`ToggleDoorAsync`(covers Open+Close)/`UseAsync` via their shared `TryBuildActionContext` context)
- [x] B.6 Permadeath entity outcome (→ `Corpse`) ships for both `PermadeathBehavior` values; `Spectate`'s session stays connected as today's default behavior (no code change needed — the session simply keeps receiving perception updates); `Disconnect`'s actual connection teardown is real hub-layer work, deferred to L.3 as planned — this slice does not silently no-op on it, it just doesn't implement the disconnect side yet
- [x] B.7 `PlayerVitalsDto` (`Aetherium.Model`) + `GameSessionManager.NotifyPlayerEventAsync` (a new player-scoped send path, independent of `MapDelta`) + `GameMapGrain.SendPlayerEventAsync`/`BuildVitalsDto`; `ReceiveDowned`/`ReceiveRespawn`/`ReceiveDied` emitted from every transition (`ResolvePlayerLethalHit` and the Downed-expiry path in B.2's tick loop)
