# com.aetherium.unity

The Unity adapter for [Aetherium](../../../README.md) — a server-authoritative multiplayer
simulation engine. One package gives any Unity game connection management, the protocol
contract, perception plumbing, and default grid/entity views; your game supplies
presentation (prefabs, VFX, audio) through a `ThemeAsset`. Full design:
[docs/design/unity-sample/unity-client-library.md](../../../docs/design/unity-sample/unity-client-library.md).

## Install

**In-repo (the Aphelion sample):** reference by relative path in `Packages/manifest.json`:

```json
"com.aetherium.unity": "file:../../../../clients/unity/com.aetherium.unity"
```

**External games:** install via git URL with a path query (no registry needed):

```json
"com.aetherium.unity": "https://github.com/danvanderboom/Aetherium.git?path=clients/unity/com.aetherium.unity#unity-client/v0.1.0"
```

**Before first use** run `.\scripts\pack-unity-client.ps1` from the repo root — it vendors
the `Aetherium.Client` core DLLs into `Runtime/Plugins/` (gitignored; see the README there).

## Quickstart

1. Add **`AetheriumClientBehaviour`** to a scene object; set the server URL (and optionally
   a world id to auto-join).
2. Create a **Theme Asset** (`Assets → Create → Aetherium → Theme Asset`) and bind your
   bundle's content ids — creature ids, terrain tile-type names, item ids — to prefabs.
   Anything unbound renders as a bright magenta capsule: loud beats invisible.
3. Add **`GridMapView`** and **`EntityViewRegistry`** next to the behaviour and assign the
   theme. Terrain reveals as you explore (remembered cells dim), entities spawn/tween/vanish.
4. Drive gameplay from your own scripts via `behaviour.Client`:

```csharp
var client = GetComponent<AetheriumClientBehaviour>().Client;
await client.Tools.MoveAsync(WorldDirection.North);   // WASD bridge: rotate + step
await client.Tools.AttackAsync(targetId);
var worlds = await client.Lobby.ListWorldsAsync();
```

All package events (`FrameReceived`, `EntityAppeared/Moved/Vanished`, `Downed`, …) are
already marshalled to the main thread; tool/lobby results complete on worker threads, so
touch Unity objects only from the events.

## Layout

- `Runtime/` — `AetheriumClientBehaviour` (lifecycle + main-thread event surface),
  `MainThreadDispatcher`, `GridMapView`, `EntityViewRegistry`, `ThemeAsset`.
- `Runtime/Plugins/` — vendored core DLLs (generated by the pack script).

The heavy lifting — mirror DTOs, the perception store with anchoring/frame-diffing/memory,
typed tool wrappers — lives in the pure-.NET [`Aetherium.Client`](../../../Aetherium.Client/)
core, tested against the real server by
[`Aetherium.Client.Tests`](../../../Aetherium.Client.Tests/) (protocol drift + in-proc
integration suites).
