# Aphelion — Asset Sourcing Plan

*Part of the [Unity sample design suite](README.md). Status: first slice committed at `samples/unity/Aphelion/` (2026-07-16); this doc is the plan of record for everything visual/audible the game needs.*

The question this answers: **where does a complete, free, repo-committable asset set come from?** The strategy is three sources, in priority order:

1. **CC0 packs** for everything organic to find (creatures, props, SFX) — primarily [Quaternius](https://quaternius.com/) and [Kenney](https://kenney.nl), the two most prolific CC0 game-asset authors.
2. **Made in-project** for everything that must fit *this* game exactly (the modular station kit, materials, VFX, UI) — the low-poly art direction makes this realistic.
3. **Generated by committed code** for music and synthetic SFX — zero licensing surface, deterministic, forever regenerable.

Hard rule (enforced via `ATTRIBUTIONS.md` review convention): **only CC0 or repo-generated content is committed.** "Royalty-free" packs that forbid file redistribution (most commercial SFX libraries, e.g. Sonniss) are usable in *builds* but never in *source* — so they're out entirely.

## Source 1 — CC0 packs (what & where)

| Need | Pack | License | Fit |
|---|---|---|---|
| Creature cast + player | Quaternius robot/space/alien models (via their [poly.pizza catalog](https://poly.pizza/u/Quaternius) — direct GLB downloads, per-model license shown) | CC0 1.0 | Excellent — flying drones, armed robots, mechs, aliens, astronauts map 1:1 onto the [game-design](game-design.md) cast |
| Sci-fi props | Same catalog (computers, crates, containers, pods, pipes, batteries) | CC0 1.0 | Good for dressing; station *structure* is better made in-project |
| Planets (window backdrops) | Same catalog | CC0 1.0 | Placeholder until a custom skybox shader (M1) |
| SFX palette | [Kenney Sci-Fi Sounds](https://kenney.nl/assets/sci-fi-sounds) (70 sounds) + Kenney [Impact](https://kenney.nl/assets/impact-sounds)/[Interface](https://kenney.nl/assets/interface-sounds) packs if needed | CC0 1.0 | Excellent — doors, lasers, impacts, force fields, engines |
| **Animated characters** | Quaternius **Ultimate Space Kit** — its download is a **public Google Drive folder** (headless-fetchable, unlike itch): self-contained glTFs with full skeletal clip sets (astronauts, mechs, enemies) | CC0 1.0 | Excellent — committed: two 18-clip astronaut player skins, the 17-clip mech (Overseer), the flying enemy (Vent Lurker) |
| Bigger structure kits (if wanted later) | Quaternius **Modular Sci-Fi Megakit** (277 pieces, Unity-URP-ready) — free zip via [quaternius.com](https://quaternius.com/) / itch.io | CC0 1.0 | The fallback if the in-project kit route underdelivers; the itch download is browser-interactive, so it's a manual grab-and-curate step |
| Textures/HDRIs (if needed) | [Poly Haven](https://polyhaven.com/), Kenney Prototype Textures | CC0 | Graybox + detail textures |

**The committed first slice** (see [ATTRIBUTIONS.md](../../../samples/unity/Aphelion/ATTRIBUTIONS.md) for per-file provenance): 14 Quaternius models covering the *entire creature cast* (Reclaimer astronaut, flying scrap-mite drone, animated custodian robot, armed sentinel, alien vent-lurker, overseer mech), six props, two planets (~3.4 MB), plus 18 Kenney sounds mapped to game events (~1.4 MB). Every model's license was verified on its source page at download time.

## Source 2 — made in-project

Per the [art direction](art-audio.md), these are authored in Unity (ProBuilder/Blender) rather than sourced, because they must obey the game's grid, palette, and emissive language:

- The **modular station kit** (~14 meshes: floors, walls, bulkhead door, window wall, lift pad, docking set) — the single most important visual asset, and precisely the thing generic packs never get right for a grid game.
- All **materials** (the three emissive accent families), **VFX** (lamp cone, bursts, beams, stasis dome), **shaders** (IR mode, sonar ping, fog-of-war memory), and **UI**.

## Source 3 — generated by committed code

Music is the *signature* problem for free assets: CC0 music that fits a specific game's identity barely exists, and CC-BY (Kevin MacLeod et al.) violates the repo rule. So Aphelion's music is **synthesized by code we own**: [`Tools~/AudioGen`](../../../samples/unity/Aphelion/Tools~/AudioGen/Program.cs), a ~350-line deterministic C# synthesizer (`dotnet run -- ../../Assets/Audio`), already committed and run. It produces:

- **`aphelion-theme.wav`** — the melodic signature (30 s): a lonely lead over a D-minor i–VI–III–VII progression with fading vibrato and ping-pong echo — "rising far, falling home," the extraction theme.
- **`pad-explore.wav` / `arp-tension.wav` / `pulse-combat.wav`** — the three adaptive-music stems (21.3 s seamless loops, wrapped-tail rendering), all on one 90 BPM D-minor grid so the [music state machine](art-audio.md#adaptive-music) can crossfade them freely.
- Body SFX perception can't buy: heartbeat (down-state), pickup chime, UI blip.

Why this is the right call beyond licensing: the stems are *parametric*. Tuning the game's tempo, key, progression, or adding a stem (the overseer choir layer) is a code change with a diffable history — music iterates like everything else in this engine's data-driven culture. If/when a human composer joins, their stems replace the generated ones under the same rules (they'd be original work, committed with provenance).

## Format & pipeline notes

- Models arrive as **GLB**; the project manifest includes **glTFast** (`com.unity.cloud.gltfast`), Unity's official glTF importer — models import on first open with no conversion step.
- Generated audio is **WAV at 24 kHz** (source-asset form; Unity transcodes to Vorbis in builds via import settings). Kenney sounds stay in their shipped OGG form.
- Committed asset weight so far: **~13 MB**. Budget ceiling before revisiting Git LFS: ~100 MB (per [repo-structure.md](repo-structure.md#source-control-hygiene-for-unity-content)).
- Everything enters through one gate: a row in `ATTRIBUTIONS.md` in the same commit — CC0 source link, or "generated by `Tools~/AudioGen`".

## Gaps this plan leaves open (deliberately)

- ~~Animations~~ **Resolved:** the poly.pizza GLBs turned out to be static exports (no rigs), but the Ultimate Space Kit's Drive distribution ships the same characters *with* full skeletal clip sets — four animated characters are now committed (`Assets/ThirdParty/Quaternius/Animated/`). Machines (mite/custodian/sentinel) animate procedurally **by design** — see the [animation direction](art-audio.md#animation-two-motion-languages).
- **A proper skybox** (planet shader with atmosphere + starfield) replaces the placeholder planet models at M1.
- **Voice/creature vocalizations:** M0 uses pitched Kenney sounds; a synthesized vocalization pass (same AudioGen approach — formant chirps for drones) is an M1 nicety.
